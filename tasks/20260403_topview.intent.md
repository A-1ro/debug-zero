# #15 TopView の実装

> 作成日: 2026-04-03

## 実装意図

`/` ルートに、ルーム作成・参加の UI を実装する。
`client:create_room` はメッセージ型に存在しないため、Create Room ボタンでは `crypto.randomUUID()` を生成して `/room/<uuid>` へナビゲートするだけ。WebSocket 接続・`client:join_room` 送信は RoomView 側で行う。

## 合意した前提

- [YES] TopView 自体は WebSocket を使わない。ナビゲーションのみ担当する
- [YES] client:create_room はメッセージ型に存在しない。Create Room ボタンで crypto.randomUUID() を生成し navigate("/room/<uuid>") する
- [YES] Join Room は navigate("/room/<roomId>") するだけ。roomId・playerName・role は URL state として渡し、RoomView が受け取る
- [YES] maxPlayers は現行の JoinRoomPayload に含まれない。Create Room パネルの Max Players フィールドは UI のみ実装し、サーバへ送信しない
- [YES] CSS は CSS Modules（TopView.module.css）＋ グローバル変数ファイル（src/client/styles/global.css）の構成。Tailwind は導入しない
- [YES] Google Fonts（Share Tech Mono, Orbitron）は index.html の <link> タグで読み込む
- [YES] グリッド背景・スキャンライン・コーナー装飾などのビジュアル演出は設計 HTML に忠実に実装する
- [YES] バリデーション: playerName は空文字不可、Join Room の roomId は空文字不可。それ以上はやらない
- [YES] デザインの忠実再現 > コードの簡潔さ。CSS 変数を使い 01-top.html の見た目を再現する
- [YES] アニメーション（pulse dot, btn hover shimmer）は CSS で実装する
- [YES] navigate() は react-router-dom の useNavigate を使用する
- [YES] フォームの state は useState で管理。サーバへのリクエストは発生しない
- [YES] モバイルレスポンシブ対応は今回やらない（1440px 固定想定）
- [YES] Max Players・Rule Set の値をサーバに送信しない
- [YES] エラー表示は form 内 inline（バリデーション失敗時のみ）

## 非対応事項

- モバイルレスポンシブ対応
- maxPlayers・ruleSetId のサーバ送信
- client:create_room メッセージ

## リスクメモ

- RoomView が navigate state から playerName・role を受け取る設計にする。インタフェース: `{ playerName: string; role: "player" | "spectator" }` を state として渡す
