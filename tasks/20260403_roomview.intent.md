# #16 RoomView (ロビー) の実装

> 作成日: 2026-04-03

## 実装意図

`/room/:roomId` のロビー画面を実装する。`useWebSocket` で接続・`client:join_room` を送信し、`useGameState` でメッセージを処理。ルームステータス（waiting → strategy-selection → in-session）に応じて UI を切り替え、`server:game_started` 受信時に `/room/:roomId/game` へ自動遷移する。

## 合意した前提

- [YES] useLocation().state を RoomNavigateState | null にキャストして playerName・role を取得。null の場合は空文字・"player" をフォールバックにする
- [YES] useWebSocket は mount 時に接続 → client:join_room を自動送信（#14 実装通り）
- [YES] state.room が server:room_updated で更新される。RoomView はこれを購読してレンダリングする
- [YES] ホストの判定は room.players[0].id === playerId（最初に参加したプレイヤー）とする
- [YES] room.selectedStrategies は全員分来るが、UI では自分の選択のみ戦略名を表示し、他プレイヤーは「SELECTED」か「—」のみ表示する（"Secret" 仕様）
- [YES] server:game_started 受信時に navigate("/room/:roomId/game") する。useEffect で state.game を監視する
- [YES] 戦略リストは basic.yaml 上位8種（Aggro, Control-Add, Control-Sub, Control-Div, Control-Mul, Hack, TrickStar, Zero）をクライアントにハードコードする
- [YES] Activity Log はローカル useState<string[]> で管理し、受信メッセージから生成する
- [YES] CSS は RoomView.module.css で実装。global.css の変数を使う
- [YES] 観戦者（role === "spectator"）の場合、Ready ボタンと Strategy 選択は非表示にする
- [YES] client:ready 送信 → server:room_updated が全員に来る
- [YES] client:select_strategy 送信 → server:room_updated が全員に来る
- [YES] client:start_game 送信 → server:game_started が全員に来る
- [YES] client:leave_room 送信後 / に navigate する
- [YES] チャット機能は今回実装しない。Activity Log は受信イベントのみ
- [YES] モバイルレスポンシブ対応は今回やらない
- [YES] 観戦者リスト表示は今回やらない

## 非対応事項

- チャット（client:chat）
- モバイルレスポンシブ
- 観戦者スロット表示

## リスクメモ

- server:game_started 監視は useEffect で state.game を依存にする。navigate を deps に含めても問題なし（useNavigate は安定参照）
- TopView からの navigate state が null の場合（直接URLアクセス）でも動作すること
